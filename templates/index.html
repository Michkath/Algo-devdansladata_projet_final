Architecture du pipeline
01
Python · BeautifulSoup
Scraping
Collecte brute des données hébergements depuis les sources officielles via scrapping.py.
02
Python · Pandas
Nettoyage
Normalisation, déduplication et validation des données dans cleaning.py. Stockage brut dans MongoDB.
03
PostgreSQL · Star Schema
Data Warehouse
Chargement dans un DW structuré (tables : hébergements, localisation, type, classement) via dw_loader.py.
04
Flask · REST
API REST
Exposition des données via endpoints filtrables, paginés et triables. Dockerisé avec docker-compose.

GET /hebergements Liste paginée & filtrable

Retourne la liste des hébergements avec support de filtres, pagination et tri.

ParamètreTypeDéfautDescription
villestring—Filtrer par commune (ex: Paris)
typestring—Type d'hébergement (ex: Hôtel)
pageint1Numéro de page
per_pageint20Éléments par page
sort_bystring—Champ de tri (ex: nom_commercial)
sort_orderstringascasc ou desc
GET /hebergements?ville=Paris&type=Hôtel&page=1&per_page=10

{
  "page": 1,
  "per_page": 10,
  "total": 120,
  "hebergements": [ { ... } ]
}
GET /hebergements/:id Détail par identifiant

Retourne les informations complètes d'un hébergement par son identifiant unique.

GET /hebergements/1

{
  "id": 1,
  "nom_commercial": "Hotel Parisien",
  "ville": "Paris",
  "type_hebergement": "Hôtel",
  "classement": "3 étoiles",
  "capacite_accueil": 50,
  "nombre_chambres": 25,
  "classement_proroge": false
}
💡 Note : Les filtres ville et type sont insensibles à la casse (ILIKE). La pagination et le tri sont compatibles avec tous les filtres.

Architecture Star Schema — La table hebergements est la table de faits centrale. Les tables localisation, type_hebergement et classement sont des dimensions. Des index sont créés sur commune, typologie et classement pour optimiser les performances des requêtes analytiques.

Documentation API REST – Hébergements Tourisme

Base URL : http://127.0.0.1:5000/

1️. Endpoint de test

GET /ping

Vérifie que le serveur tourne.
Réponse :
{
  "status": "ok",
  "message": "pong"
}
2️. Liste complète des hébergements

GET /hebergements

Retourne tous les hébergements de la base.
Paramètres optionnels pour filtrer et paginer :
| Paramètre | Type | Description |
|-----------------|--------|-------------|
| ville | string | Filtrer par commune (ex: Paris) |
| type | string | Filtrer par type d’hébergement (ex: Hôtel) |
| page | int | Numéro de page pour pagination (défaut 1) |
| per_page | int | Nombre d’éléments par page (défaut 20) |
| sort_by | string | Champ pour tri (ex: nom_commercial) |
| sort_order | string | asc ou desc |

Exemples :

GET /hebergements?ville=Paris&type=Hôtel&page=1&per_page=10&sort_by=nom_commercial&sort_order=asc

Réponse :

3️. Détail d’un hébergement

GET /hebergements/<id>

Retourne les détails d’un hébergement précis par son id.
Exemple :
GET /hebergements/1

Réponse :

{
  "id": 1,
  "nom_commercial": "Hotel Parisien",
  "site_internet": "https://hotelparis.com",
  "ville": "Paris",
  "code_postal": "75001",
  "type_hebergement": "Hôtel",
  "capacite_accueil": 50,
  "nombre_chambres": 25,
  "classement": "3 étoiles",
  "date_classement": "2023-06-19",
  "classement_proroge": false,
  "adresse": "12 Rue de Paris",
  "mention": null
}

4️. Filtres dynamiques
Tu peux combiner ville et type pour filtrer.
La pagination et le tri sont compatibles avec les filtres.

Exemple :

GET /hebergements?ville=Lyon&type=Village&page=2&per_page=5&sort_by=capacite_accueil&sort_order=desc

5️. Notes
Tous les champs liés aux dimensions (localisation, type_hebergement, classement) sont déjà liés via leurs id.
classement_proroge : true = "oui", false = "non".
